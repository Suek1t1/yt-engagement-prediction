## 2026-06-30（第11回）

### 今日やったこと
- 第10回の「次にやること」のうちモデル改善2件に集中。
- 4段階分類の「低」段階の崩れを、ビン境界の切り直しで改善（`likes_4class_v2.py`）。
  4方式を比較: v1=学習分位点 / A=全データ分位点 / B=対数等幅 / C=チャンネル相対比。
- 最低保証モデルを「予測時に下限段階を出力する実用関数 `predict_floor()`」に仕上げた
  （`likes_floor_predict.py`）。conf（安全率）で保証の堅さと情報量を調整可能に。
- 追加検証: チャンネルの知名度（学習期間のトレンド入り本数）で test を層別し、
  予測しやすさを比較（`likes_fame_split.py`）。有名(6本+)/中堅(2-5)/無名(1)/新規。

### 分かったこと / 結果
- **「低」段階が崩れる正体は分布シフト**：test期間のlikes中央値(20,562)は学習期間
  (5,642)の約3.6倍。likes絶対値で境界を引くと未来では大半が上位ビンに寄り、
  「低」が痩せる（v1: support=15, F1=0.125）。**ビンの引き直しだけでは根治しない**。
- **4方式の比較（test）**:
  - v1 学習分位点: 正解率0.616 / 低F1=0.125（support 15）
  - A 全データ分位点: 正解率0.567 / 低F1=0.282（support 37）— 低は改善も全体↓
  - B 対数等幅: 正解率0.767 / 低F1=0.000（support 1）— 全体は高いが低が消滅
  - **C チャンネル相対比**: 低F1=0.320（support 227）で「低」を最も復活。ただし
    全体正解率0.344。問題が「実力比でどれだけ伸びたか」に変わるため難化する。
  → 「低」を救うにはターゲット定義をチャンネル相対比に変えるのが最有効。
    絶対数の4段階を保つなら A（全データ分位点）が無難な妥協点。
- **最低保証の実用化**：`predict_floor()` が新規動画→(下限段階・likes下限値・保証成立率)
  を返す。conf を上げるほど控えめ＝保証↑情報量↓。
  - argmax→0.851 / conf0.5→0.881 / **conf0.7→0.950** / conf0.9→0.994。
  - デフォルトconf=0.7で段階別保証: 低1.00 / 中低0.98 / 中高0.94 / 高0.91。
  - サンプル3動画のデモは全て保証成立（控えめな下限に実際が収まった）。
- **チャンネル知名度別の予測力（fame_split / Spearman・AUC）**:
  - 中堅(2-5本) が最も予測しやすい（ρ=0.751, AUC=0.947）。実績が安定し素直に効く。
  - 有名(6本+) は ρ=0.690 / AUC=0.883。話題性で上下にブレるため中堅に劣る。
  - 無名(1本) は ρ=0.588 / AUC=0.790 と低下。
  - **新規(学習期間に未出現) はほぼ予測不能**（ρ=0.125, AUC=0.602≒コイン投げ）。
    → 「誰が出したか」が支配的で、それが不明な新規chでは投稿前予測が崩壊する、
       という従来知見をチャンネル軸で定量的に裏付け。
  - content(内容のみ)モデルは新規群で full(0.125)を上回る(0.281)。実績がノイズに
    なる新規chでは内容特徴の方がまだマシだが、どちらも実用水準には届かない。

### つまずき・疑問

## 2026-06-25（第10回）

### 今日やったこと
- likes予測プロジェクト全体をレビュー。
- カテゴリーのまとめ方を検討し、n<500統合版・意味でまとめる案A(6グループ)/案B(8グループ)を実装。
- 改善案1〜4（リーク排除・評価の信頼性・ablation・モデル比較）をコードで検証。
- リークなし正式版モデル `likes_pretime_rf.py` を作成。
- 評価を時系列分割に変えた `likes_pretime_timesplit.py` を作成。
- 今後の方向性を3つ（精度/分析/問題設定変更）すべてシンプルに検証。
- 「事前確定情報でどこまで細かく刻めるか」を順位予測・多クラス分類で探索。
- 4段階分類の正式モデルを作成。
- 予測を「最低保証（実際≥予測で正解）」として使う設定を検証・正式化。

### 分かったこと / 結果
- **当初のR²=0.99は「逆算」だった**：説明変数 views/dislikes/comment_count は likes と同時に決まる量（リーク）。特徴量重要度の99%を占めていた。
- **ablation**：実スケールR²を上げているのは主に「ジャンル分割」。対数変換は実R²をほぼ上げず、対数R²とMAPE/RMSEを改善する（効く指標が違う）。
- **リークを断つと**：投稿時点特徴量だけでR²≈0.95。ただし重要度の0.84はチャンネル過去平均likesに集中＝「誰が投稿したか」がほぼ全てを決める。
- **チャンネル特徴のリークに注意**：全データ平均だと漏れる。学習fold限定で算出すべき（→正式版で対応）。
- **最大の発見＝3段階のリーク**：当初R²=0.99は ①同時生起変数 ②動画重複（37,236行→ユニーク5,817件）③ランダム分割による未来情報混入、の積み重ね。すべて排して時系列評価すると **真のR²≈0.40（±0.20）**、対数R²はほぼ0。
- 結論：**投稿前のlikes予測は実際には難しい**。これは失敗ではなく、評価設計の重要性を示す誠実な結果。

#### 後半（方向性の探索）で分かったこと
- **3方向の探索**：①TF-IDFテキスト特徴を足すとR² 0.41→0.45に向上（テキストは効く）。②チャンネル実績を除くとR²がマイナス＝「内容より発信者」が決定的。③問題設定を変えると、エンゲージ率予測はR²≈0.43、**「伸びるか否か」の二値分類はAUC 0.82**と実用的。
- **カテゴリ別の予測しやすさが激しく違う**：Sports(0.87)/Education(0.79)は当てやすく、Entertainment(-0.11)/Gaming(-0.47)は予測不能。
- **刻み幅の限界**：ピタリ正解率は3段階69% / 4段階62% / 5段階52% / 10段階32%。±1段階許容なら3段階97%・4段階91%・5段階88%。順位予測のSpearman ρ=0.62。→ 4〜5段階が刻みと精度のバランス点。
- **4段階分類（正式版）**：正解率61.6%（ランダム25%）、±1段階91.1%。「高」段階はprecision0.76と良いが、「低」段階はトレンド動画に低likesが少なく崩れる。TF-IDF全体の重要度0.51でテキストの寄与が大きい。
- **最低保証という使い方**：予測段階を「下限」とみなし実際≥予測を正解とすると、成立率は3段階88% / 5段階83% / 10段階76%と、刻んでも落ちにくい。特に下位段階を下限にすれば10段階でも99〜100%保証。「最低でもこの段階は伸びる」という控えめな約束として有用。

### つまずき・疑問
- MAPEが539%になる → likes2桁以下の動画（約990件）でパーセント誤差が爆発するため。主指標はR²/MAEにし、MAPEは小likes動画で不安定と注記する。
- 検証時、計算が重く木の本数を一部40本に削減（結論は不変）。最終報告用は100〜200本で回し直す。
